Итак, если ты запустил программу (если у тебя руки не из жопы и ты знаешь как это делать), то перед тобой
предстанет главное окно программы, которое содержит в самом верхнем блоке заголовок(название программы)
и меню.

Чуть ниже меню расположен блок "Account settings".
Он задает один из трех доступных режимов атаки:
1.Список пользователей + один пароль.
Чтобы назначить список пользователей необходимо кликнуть на поле слева и в появившемся окне открыть
txt документ со списком пользователей.
2.Список пользователь:пароль.Данный список загружается аналогичным образом, что и в первом случае.
Разделителем в нем служит двоеточие (:), которое, собственно, отделяет список пользователей от паролей.
3.Пользователь + список пароль.
Этот режим используется, если нужно получить доступ к конкретному пользователю, чуть ниже расположен
чекбокс.В случае, если он отмечен и в процессе атаки будет найден верный пароль, то атака будет остановлена
автоматически.

Блок "Statistic"
Этот блок отображает подробную статистику работы программы.
Сверху вниз в нем расположены элементы, каждый из которых имеет свое предназначение.
Первый опреляет кол-во пользователей/паролей (в зависимости от выбранного режима), которые будут обработаны.
Второй определяет кол-во пользователей/паролей, которые осталось обработать.С каждым запросом уменьшается.
Третий содержит кол-во пользователей/паролей, которые уже обработаны.
Четвертый содержит кол-во пользователей/паролей, для которых были найдены успешно сответствующие комбинации.
Пятый содержит кол-во пользователей/паролей, для который обработанные комбинации оказались неудачными.
Два последующих элемента пока что не несут практической пользы.
Самый нижний - шкала, которая отображается в процентах работу программы (от нуля до ста).

Вернемся к меню.
Пункт меню "Action" содержит в себе следующие элементы:
"Start" - начать атаку
"Pause" - приостановить атаку
"Continue" - продолжить атаку, если она была приостановлена
"Stop"  - остановить атаку
"Exit"  - принудительный выход из программы с прерыванием всех рабочих процессов.

Пункт меню "Main Setting"
Юзабельны в нем только два:
"Attack configuration".
Здесь задается страница авторизации, передаваемые параметры, а так же режимы определения
успешности.Рабочие конфигурации приложены к программе.Они представляют из себя структурированные "ini" файлы.
Данные настройки являются основными и если их не задать, то программа не будет просто напросто работать.
"Network settings".
"Threads" определяет кол-во одновременных соединений к серверу.
"Timeout" read определяет задержку между соединениями.По умолчанию установлен в 10000 милисекунд (10 секунд).

Пункт меню "Logs"
Содержит два элемента:
"Show" - отобразить
"Hide" - скрыть

Логи, которые заполняются всеми успешными и неудачными попытками авторизации в процессе атаки на сервер.
Так же, если сервер перестанет отвечать, то атака будет автоматически остановлена и туда будет добавлено соответствующее
сообщение.
Окно с логами можно очистить и упорядочить с помощью кнопок "Clear" и "Sort" соответственно.
Упорядочивание работает таким образом, что все статусы Bad будут вверху списка, а внизу будут те, что имеют статус Good.

Про "Help" не буду расписывать, тут все итак понятно (=
